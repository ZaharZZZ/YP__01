	каждый метод начинается с "public static"
	в методе:
			при необходимости переменные и тд (типа List<> list)
		открываем подключение using(MySqlConnection conn....)
		открываем соединение conn.Open();
		создаём строку sql-запроса (SELECT ... FROM ... ) доп условия если надо WHERE и тд.
		создаём команду using/мб без юзинга(MySqlCommand command....)
			если нужно, добавляем параметры command.Parameters.AddWithValue("@параметр", переменная парметра)
		создаём чтение данных и получаем их в чтение через команду using(MySqlDataReader reader = command.ExecuteReader())
		**для SELECT используют ExecuteReader
		**для INSERT/UPDATE/DELETE используют ExecuteNonQuery
		
		Варианты
			1 - если ищем и/или выбираем/проверяем что-то конкретное, то в запросе у нужных параметров ставим @
				например: WHERE login = @login AND password = @password
			дальше проверяем есть ли запись/нужное if (reader.Read()), если true то возвращаем/добавляем 
			необходимый объект, и извлекаем данные из ридера, например: id = reader.GetInt32("id"),
			если false, то возвращаем null и идём нахер
			*** в этом случае параметры command нужны скорее всего будут

			2 - если нужно просто получить данные, то @ ставим перед всем запрсом @"SELECT ..." 
			***параметры command не нужны
			дальше читаем в цикле данные while(reader.Read()) и создаём объект и добавляем куда надо, а внутри
			объекта извлекаем данные из ридера, например: article = reader.GetString("article")
			возвращаем список, или чё надо было
